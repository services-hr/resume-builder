# スプレッドシート設計 & HubSpot取り込み手順

このドキュメントは、候補者データの中央データベースとなる**Googleスプレッドシート**の設計と、そこから**HubSpotにCSVインポート**する手順をまとめたものです。

---

## スプレッドシートの役割

```
ビルダーUI → GAS Webhook → PDF生成（Drive保存） + シートに1行追加
                                              ↓
                                        HubSpotにCSVで取り込み
```

---

## ステップ1：スプレッドシートを作成

### 1-1. 新規スプレッドシート作成

1. https://sheets.google.com にアクセス
2. 「空白のスプレッドシート」をクリック
3. ファイル名を「**候補者データベース**」に変更
4. URLから**スプレッドシートID**をメモ：

```
https://docs.google.com/spreadsheets/d/【このIDをメモ】/edit
```

### 1-2. シート名を決める

- デフォルトで「シート1」というシートが作られます
- これを右クリック → 名前を変更 → `candidates` に変更

（※違う名前にしてもOK。その場合はGAS側の `SHEET_NAME` プロパティを合わせる）

### 1-3. 列ヘッダーの自動生成

GASの `setupSheetHeaders()` 関数を実行すると、1行目に31列分のヘッダーが自動で書き込まれます。

**手順：**
1. GASエディタを開く
2. スクリプトプロパティに `SPREADSHEET_ID` と `SHEET_NAME` を設定（セットアップガイド参照）
3. 関数選択プルダウンで `setupSheetHeaders` を選択
4. ▶ 実行
5. シートを開いて、1行目に列ヘッダーが緑色の背景で入っていれば成功

## ステップ2：HubSpot側のカスタムプロパティ作成

### 2-1. カスタムプロパティ作成手順

1. HubSpotにログイン
2. 右上の歯車アイコン（設定）
3. 左メニュー「データ管理」→「プロパティ」
4. 「オブジェクト種類を選択」→ **コンタクト**
5. 右上「プロパティを作成」

## ステップ3：スプレッドシートからCSVをエクスポート

### 3-1. 全データをエクスポートする場合

1. スプレッドシートを開く
2. メニュー「ファイル」→「ダウンロード」→「**カンマ区切り形式（.csv）**」
3. CSVファイルが自動ダウンロードされる

### 3-2. 前回インポート以降の新規データだけエクスポートする場合

毎月1回など定期インポートする運用の場合、**`submitted_at` 列で絞り込んだ別シート**を作ると便利です。

1. 新しいシートを追加（例：`export_2026_04`）
2. A1セルに以下を入力：

```
=QUERY(candidates!A:AF, "SELECT * WHERE A >= date '2026-04-01' AND A < date '2026-05-01'", 1)
```

3. 条件に合う行だけが表示される
4. このシートを選択した状態でCSVエクスポート

## ステップ4：HubSpotにCSVインポート

### 4-1. インポート開始

1. HubSpotにログイン
2. 上部メニュー「コンタクト」→「コンタクト」
3. 右上「**インポート**」ボタン
4. 「**インポートを開始**」→「**コンピューターからファイル**」→「次へ」
5. 「**1つのファイル**」→「次へ」
6. 「**1つのオブジェクト**」→「次へ」
7. 「**コンタクト**」を選択 →「次へ」

### 4-2. CSVファイルをアップロード

1. 3-1または3-2でダウンロードしたCSVファイルを選ぶ
2. 「次へ」

### 4-3. 列のマッピング

HubSpotが自動で列名を見て、標準プロパティに自動マッピングしてくれます。カスタムプロパティも、事前に 2-2 で作った内部名と一致していれば自動的にマッピングされます。

マッピング画面で以下を確認：
- ✅ `lastname` → 姓（HubSpot標準）
- ✅ `firstname` → 名（HubSpot標準）
- ✅ `email` → メールアドレス
- ✅ `phone` → 電話番号
- ✅ `current_company` → 会社
- ✅ `jobtitle` → 役職
- ✅ カスタムプロパティも順次自動マッピング

取り込まない列（`submitted_at`, `source`, `pdf_file_id` など）は「**列をスキップ**」を選択。

### 4-4. 重複処理の設定

「**このファイルに既存のコンタクトを更新するレコードが含まれていますか？**」

- メールアドレスで一意に管理する場合：「はい」→ `email` 列を重複判定キーに指定
- 毎回新規として登録する場合：「いいえ」

### 4-5. インポート実行

1. 右下「**インポート終了**」をクリック
2. 数分後にHubSpotから完了メールが届きます
3. コンタクト一覧で新規レコードを確認

## 運用のコツ

### 月次インポートのルーチン化

1. **月初**：先月分の `submitted_at` でQUERYしたシートからCSVをダウンロード
2. **HubSpot**：CSVをインポート、重複判定キーは `email`
3. **ファイル管理**：過去CSVは Google Drive の別フォルダにアーカイブ

### 重複の防止

- 同じ人が複数回ビルダーを使うと、スプレッドシートに複数行追加されます
- HubSpotのインポートで `email` を重複キーにすれば、**最新データで既存レコードが更新**されます
- スプレッドシート側には全履歴が残り、HubSpot側には最新版だけが反映される、という綺麗な状態になります

### PDF Driveリンクを HubSpot に表示

HubSpot の `pdf_drive_url` プロパティは「1行テキスト」で作っておくと、コンタクトの詳細画面でクリック可能なリンクとして表示されます。より見栄えを良くしたい場合は、HubSpot の URLプロパティ型を使ってもOKです。

## トラブルシューティング

### Q: 列のマッピングが自動でされない
A: CSVの1行目の列名（内部名）が、HubSpot側のカスタムプロパティの**内部名（Internal name）**と完全一致しているか確認してください。ラベルではなく**内部名**です。

### Q: 日本語が文字化けする
A: CSVをダウンロードする際、Google スプレッドシートは UTF-8 で出力しますが、Excelで開くと化けることがあります。HubSpotは UTF-8 を正しく解釈するので、Excelで開かずに直接HubSpotにアップロードしてください。

### Q: 複数行テキスト（`career_history_all` など）に改行が入っている
A: これはCSV規格上、ダブルクォートで囲まれたフィールド内の改行として正しく扱われます。HubSpot側でも改行付きで保存されます。

### Q: HubSpot無料プランでも使える？
A: CSVインポートはHubSpot無料プランでも使えます。コンタクトは無料プランで **100万件** まで保存できます。カスタムプロパティも無料プランで使用可能です（ただし無料プランでは一部のプロパティタイプ・設定に制限があります）。
